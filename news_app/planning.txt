1. Functional Requirements

User Management
    Users must be able to register for an account.
    Users must be able to log in and log out.
    Users must have roles:
        Reader
        Journalist
        Editor

Article Management
    Journalists can't:
        Create articles
        Edit their articles
    Articles must contain:
        Title
        Content (description)
        Author
        Publisher
        Date created
        Approval status

Article Approval
    Editors must be able to:
        View all unapproved articles
        Approve articles
        Decline articles
Article Viewing
    Readers must be able to:
        View all approved articles
        View articles from subscribed journalists
        View articles from subscribed publishers

Subscription System
    Readers must be able to:
        Subscribe to journalists
        Subscribe to publishers
        Unsubscribe from journalists or publishers

Notifications
    When an article is approved:
        All subscribers of the journalist or publisher must receive an email notification.
REST API
    The system must provide a RESTful API that allows external clients to:
        Retrieve approved articles
        Retrieve articles from subscribed journalists
        Retrieve articles from subscribed publishers

Newsletter System
    Editors or journalists can create newsletters
    Newsletters can contain multiple articles


2. Non-Functional Requirements

Security
    Users must be authenticated before accessing protected pages.
    API endpoints must require authentication.

Performance
    Article retrieval should be efficient using database filtering.
    API responses should return structured JSON data.

Scalability
    The system must support multiple users, articles, and publishers.
    Database
    The application must use MariaDB as the database.

Maintainability
    The system should follow a modular architecture using Django apps.
    Code should be structured using:
        Models
        Views
        Templates
        Serializers
        APIs

Documentation
    The project must include a README.md explaining:
        Installation
        Project structure
        API usage
        Setup instructions

3. Front-End Planning

Home Page
    Purpose:
        Entry point for the application.
    Features:
        Login button
        Signup button

Dashboard
    Purpose:
        Central navigation area after login.
    Reader actions:
        View Articles
        View Subscribed Articles   
    Journalist actions:
        Create Article
    Editor actions:
        Approve Articles

Article List Page
    Displays:
        Article title
        Author
        Publisher
        Article content
    UI elements:
        Article cards
        Subscribe buttons for journalists

Create Article Page
    Form fields:
        Title
        Content
        Publisher
    Validation:
        Title and content are required.

Approve Articles Page
    Editors can:
        View pending articles
        Approve articles
        Decline articles
